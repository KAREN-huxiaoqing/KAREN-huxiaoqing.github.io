浅析SEO

   首先，要进行SEO，我们要来了解一下，搜索引擎工作原理。

搜索引擎工作原理

  • 爬行和抓取：搜索引擎派出spider去访问互联网上的网页，跟踪网页的链接，爬取数据抓取文件，存入数据库；
  • 建立索引：在数据库中，spider抓取的页面文件分解、分析，并以巨大表格的形式存入数据库，形成索引
  • 搜索词处理：搜索引擎程序对搜索词进行分词，整合，检查拼写错误，错别字等处理
  • 排序：搜索引擎从数据库找出符合搜索词相关的网页，根据排名算法进行排序，并返回搜索页面


那么，什么是SEO？

SEO(Search Engine Optimization)

搜索引擎优化，是指对网站内部及外部进行优化，增加网页在搜索引擎自然搜索结果中的收录数量以及关键词排名。

SEO是一个很大的范围，那本篇先介绍一些
对于代码方面，SEO可以分为两部分：

  • 网页结构布局
  • 网页代码

    网页结结构布局

    • 结构布局优化：用扁平化结构（层次结构超过三层小蜘蛛就不愿意爬了）
  • 扁平化的目录层次
  • 导航seo优化（头部、底部、内容部分，主导航、副导航、分类导航，尽量用文字)
  • 网站导航与链接优化

网页代码

1.TDK优化：

TDK就是指title，description，keywords

标题 title
    Title是搜索引擎了解网页的入口，只强调重点，关键词不要重复超过两次

推荐做法

  1. 标题要主题明确，包含这个网页中最重要的内容，只放1-2个关键词
  2. 简明精练，一般不要超过25个汉字，50个英文字母
  3. 用户浏览通常是从左到右的，越核心的关键词排放位置越靠前
  4. 使用用户所熟知的语言描述。如果你有中、英文两种网站名称，尽量使用用户熟知的那一种做为标题描述
  5. 每个页面的title不要相同

推荐做法

  1. 网站首页、频道页、产品参数页等没有大段文字可以用做摘要的网页最适合使用description
  2. 准确的描述网页，不要堆砌关键词,3-4个关键词为佳，越核心的关键词越方在前面
  3. 为每个网页创建不同的description，避免所有网页都使用同样的描述
  4. 长度合理，不要超过100个汉字、200个英文字母

2.语义化标签
  • 合理利用<h1>~<h6>:
    一个网页最好只有一个<h1>,这个标签权重高，一般spider会特别关注
  • 重要的文字用<strong>
  • <img> 为图片加上alt属性，方便用户知道图片的作用，需要强调的图片加上 title属性

不显示图片的效果：

  • 可以使用h5里面的新标签,如<nav>, <header>,<footer>，减少<div>
  • <a>链接外部网站，应该添加<a href="#" rel="nofollow">这样可以是spider忽略链接内容
3.代码内容优化
  • 在布局的时候，将重要的内容放在前面
  • spider只能识别静态的网页，所以重要的内容不要用js渲染
  • display: none;会被spider忽略，谨慎使用；
  • URL优化：
    一个网站的URL越短越好，不论是内链还是外链；目录层次尽量少，文件目录使用关键词或描述性的词；全部使用小写，连词符用-较好
  • 外链CSS和JS，提升网站速度。CSS放在文件头部，JS放在文件尾部
  • CSS Sprites 使用雪碧图，压缩照片，减少http请求的次数

flexbox属性详解

   flexbox布局要在父级HTML元素上面设置display属性

flexbox的兼容性

flexbox属性详解

1.首先我们来了解flexbox,它有两部分flex containner伸缩容器和flex item伸缩项目，flexbox有一条主轴和一条交叉轴
这是一个flexbox的分解图

flex containner容器方面的属性

1.flex-direction:

主轴方向上的排列，控制每一个元素

属性：
row: 从左到右排成一列
row-reverse: 从右到左排成一列
column：从上到下排成一列，
column-reverse：从下到上排成一列

来个小例子，理解一下：

flex-direction: row效果就是这样的

flex-direction: row-reverse

同样，column可以自己去试试~

那么，如一行内容过多又怎么解决呢

2.flex-wrap:

换行属性，能减少浮动的使用

属性：
nowrap: 默认不自动换行，自动伸缩适应容器的宽度
wrap: 自动换行，不改变自身的宽度，从左到右，从上到下顺序排列
wrap-reverse: 自动换行，从左到右，从下到上顺序排列

默认值nowrap,容器中所有的item会自动适应父容器的宽度，不论是否对item设置宽高，效果如下：

flex-wrap: wrap自动换行，在父容器中均匀分布

flex-wrap: wrap-reverse

好了，每次设置两属性可能有点麻烦，那flex-flow属性就可以替代这两个

3.flex-flow:

同时设置wrap和direction属性
一般格式：
flox-flow: row nowrap默认值
direction和wrap的属性可以随意组合

要是想设置item的对齐方式，就要用的justify-content属性

4.justify-content:

属性：
flex-start：主轴线上，左对齐，默认值
flex-end:主轴线上，右对齐
center：主轴线，居中对齐
space-between： 主轴线上，设置每个item相等间距，可以理解为两端对齐
space-around： 主轴线上，设置每个item相同的间距，包括最后一个和第一个


区别space-between和space-aroud这两个属性，来个小例子理解一下
space-around每一个item之间的距离相等，准确的讲是每个item左右的空间相等

space-between可以理解为两端对齐，两端的item靠近容器内壁，剩余空间均匀分布

那控制交叉轴的属性又有哪些呢

1.align-items

交叉轴上的对齐方式

属性：
stretch：拉伸，若项目未设置高度，则每个item占满整个容器
flex-start：交叉轴上对齐
flex-end：交叉轴下对齐
center：交叉轴居中对齐
baseline：item第一行文字的基线对齐

align-items：strech若每个item有设置宽高，则显示为设置的形状；若未设置，则默认适应父容器的高度

flex-start、center、flex-end

align-item: baseline;这个属性是设置每一个item，以她的第一行文字的基准线对齐

这是针对容器中的每一个item的属性

2.align-content:

定义多跟轴线对齐方式

属性：
flex-start:交叉轴上向上对齐
flex-end:交叉轴上向下对齐
center：交叉轴居中对齐
space-between:交叉轴上两端对齐，轴线之间的距离均匀分布
space-around:每根轴线两端的间隔相等
stretch: 占满整个交叉轴,每一条轴线分配的空间是均匀的

对比一下，align-items和align-content他们的显示效果是差不多的，align-items适用于单行item，而align-content对单行item不起作用，只适用于多行

space-around与justify-content：space-around类似,h轴线两端空间均匀分布

space-between与justify-content：space-around类似，不过这是交叉轴方向，可以理解为上下对齐

flex item伸缩项目，对容器里面的每一个项目也可以进行单独的属性设置

1.order:

定义item的排列顺序，但这是主轴方向上的，数值越小越靠前，默认值为0,可以为负值
order: 0;

根据order值排序：
item2<item3=item5<item4<item1
显示的效果如下：

2.flex-grow

定义每个项目的放大比例，但注意这是分配剩余的空间，默认值为0，负值无效。

3.flex-shrink

定义每个item的缩小比例，默认值为1,负值无效。

4.flex-basis:

类似于属性width和height，定义item占据的空间,用来设置width，默认值为auto

5.flex：

这个属性是以上三个属性的简写flex: grow shrink basis;

6.align-self：

属性：
stretch：拉伸,适应父容器的height
flex-start：交叉轴上对齐
flex-end：交叉轴下对齐
center：交叉轴居中对齐
baseline：item第一行文字的基线对齐
为每个item单独设置对齐方式


对比一下，align-items和justify-content属性，控制的是父容器中的所有item的位置变化，而align-self只控制了单个的item
